# 実験の実行と結果の記録（再現性ファースト）

`experiment-plan` の計画書（`docs/plans/<ID>-<slug>.md`）に基づいて実行する。
計画書がない実験は、先に `experiment-plan` で計画を作る（場当たり実行をしない）。

本プロセスは手を動かす作業が中心のため、サブエージェントに委譲する場合は
`implementer`（下位ティア、`.claude/agents/implementer.md`）を使う。
ただし結果の解釈・レポートの考察執筆は implementer に任せず、本スキルの呼び出し元
（または `planner`）が行う。

作業単位の規則（規約参照）:
- **1 worktree = 1 issue = 1 サブエージェント**。対応 issue のない実行をしない
  （issue は計画時に `experiment-plan` が起案している前提。なければ差し戻す）。
- 並行実行は worktree で隔離する（Agent ツールの `isolation: "worktree"` か、issue 番号入り worktree）。
- スクリプトの実装・修正を伴う場合は **TDD**: 完了条件を `tests/`（pytest）の失敗するテストとして
  先に書き、通るように実装する。

## 実行前チェック（環境の再現性）

- [ ] 環境は `scripts/setup_env.sh` 由来か（uv + `requirements.txt` の固定依存。素の pip install をしない）
- [ ] 依存を追加した場合、`requirements.txt` を更新したか（推移的依存まで固定、git 依存はハッシュ固定）
- [ ] 上流パッチ（patches/）が適用された状態か
- [ ] 実行環境（OS・Python・主要パッケージのバージョン）を記録する仕組みがあるか
      （本リポジトリでは `outputs/run_meta.json` 相当に保存）

## 実行の規約

1. **スクリプトの配置**: 本系列は 00〜0N、追加実験は 10 番台、ビルドは 03。
   新実験は既存スクリプト（`scripts/10_experiments.py` 等）への追加か、次の番号で新設する。
2. **パラメータ**: シード・ε・サンプル数・ビン数等はスクリプト冒頭の定数か引数に集約する
   （本文・図のキャプションから参照できるように）。計画書の条件と一致させる。
3. **シード**: 計画書のシード集合をそのまま使う。対象間で同一集合。
   途中で増減した場合は計画書に追記する。
4. **長時間実行**: バックグラウンド実行とし、完了待ちでセッションをブロックしない。
   メモリ・キャッシュ起因の揺れ（例: JAX キャッシュ）に注意し、対象間でプロセスを分けるか明示的にクリアする。
5. **生成物**: 指標は JSON（`outputs/` `experiments/`）、図は PNG（`figures/`）に保存。
   生成物はすべて再実行で再生成可能に保つ（手作業の加工を挟まない）。
6. **失敗の記録**: 実行中に遭遇した問題（クラッシュ・数値破綻・回避策）は、その場で
   `content/engineering-notes.md` の「問題 / 原因 / 対応」テーブルに追記する。後回しにすると失われる。

## 結果の整理（レポートへの反映）

結果は記述の分離原則（`.claude/docs/documentation-conventions.md` 第 2 節）に沿って書く:

1. 数値表 + 図 + **実行条件**（データ・サンプル数・シード数・ε・依存固定への参照）をセットで書く。
2. 「所見」は結果の直後に置き、考察（推定）であることが分かる書き方にする。
   単一シードなら断定せず、複数シードなら mean±std で書く。
3. 計画書の仮説と突き合わせ、支持/棄却/判断不能を明記する。判断不能なら不足している試行を残課題へ。
4. レポート骨子の該当 TODO を消し込み、`report-skeleton` の stage を進める。
5. 計画書に実施記録（実行日・コミット・所要時間・結果の要約 1 行）を追記して閉じる。
6. 仕上げに `report-review`（数値整合）→ 必要に応じて `publish-check`（公開反映）を実行する。

## 出力形式

```
## experiment-run 結果
### 実行: 実験<ID>（計画書: docs/plans/...）
- 環境: （uv / requirements.txt のコミット / 実行時間）
- コマンドと exit code
### 結果サマリ（仮説の支持/棄却/判断不能）
### レポート反映: 更新したファイルと消し込んだ TODO
### engineering-notes への追記（あれば）
```
