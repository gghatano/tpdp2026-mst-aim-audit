# 再現手順・エンジニアリングノートのレビュー

`.claude/docs/documentation-conventions.md` の第 1・4 節を前提に、
エンジニアリング系ドキュメント（`content/setup.md`・`content/usage.md`・`content/reproduce.md`・
`content/engineering-notes.md`）と、その実体である `scripts/`・`requirements.txt`・`patches/` の整合を点検する。

## 手順

1. README のクイックスタートと `content/reproduce.md` を読み、宣言されている再現手順を把握する。
2. `scripts/setup_env.sh`・`scripts/run_all.sh`・番号付きスクリプトを読み、
   手順書とスクリプトの実体が一致しているか突き合わせる。
3. 以下のチェックリストで点検し、ドキュメント側の軽微な不一致は修正する。
   スクリプトの挙動変更は提案に留める（再現結果が変わりうるため）。
4. 環境が許せば、軽量な検証（構文チェック、`python -m http.server` でのローカル閲覧、
   ビルドスクリプトの実行など、DP 生成を伴わない範囲）を行う。

## チェックリスト

### 再現手順
- [ ] README のクイックスタート・`reproduce.md`・`run_all.sh` の 3 つで、コマンドと実行順が一致しているか
- [ ] スクリプトの番号帯（00=データ、01=生成、02=評価、10台=追加実験、03=ビルド）と
      README の構成図・各ドキュメントの説明が一致しているか
- [ ] 前提条件（OS・Python バージョン・非対応プラットフォーム）が README 冒頭で宣言されているか
- [ ] 所要時間の目安・生成物の場所（outputs/ figures/ htmls/）が書かれているか
- [ ] git 管理外ディレクトリ（data/ src/ .venv/）が .gitignore と説明の双方で扱われているか

### 依存固定・パッチ
- [ ] `requirements.txt` が推移的依存まで固定されており、git 依存はハッシュ固定か
- [ ] `patches/` の各パッチが setup スクリプトから適用されており、
      engineering-notes に「問題 → 原因 → 対応 → 副作用（例: DP 予算消費は不変）」が記録されているか
- [ ] パッチの upstream 還元（PR 提案）がバックログに載っているか
- [ ] 外部データ取得の URL・消失リスク対策（チェックサム等）の状況がバックログに反映されているか

### エンジニアリングノート
- [ ] 「研究知見ではなく実装・運用の一次記録」という位置づけが冒頭で宣言されているか
- [ ] 問題一覧テーブル（# / 問題 / 原因 / 対応）と詳細節の項目が対応しているか
- [ ] 新しく遭遇した問題（直近のコミット・セッションで対処したもの）が記録から漏れていないか
- [ ] 該当コード位置・出典タグが本体レポートの参考文献と整合しているか

## 出力形式

```
## repro-engineering-review 結果
### 検証したこと（実行した場合）
- コマンドと結果
### 修正済み（ドキュメント側の不一致）
- <file>:<line> 内容
### 要判断（スクリプト・依存に関わる指摘）
- [重要度] 指摘と提案
```
