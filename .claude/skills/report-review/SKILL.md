# 実験レポートのレビュー（前提・結果・考察・課題）

`.claude/docs/documentation-conventions.md` の第 2 節「記述の分離原則」を前提に、
レポート系 Markdown（このリポジトリでは `content/REPORT.md`・`content/EXPERIMENTS.md`・`content/method-*.md`）を点検する。

## 手順

1. 対象ファイルを読み、章構成を把握する。レポート本体の標準構成:
   アブストラクト → イントロ（貢献の箇条書き）→ 関連技術・前提 → 実験設定 → 結果 → 考察 → 残課題 → 結論 → 再現補足 → 参考文献。
2. 以下のチェックリストで点検する。
3. 表記ゆれ・リンク切れ・数値転記ミスなどの軽微な問題はその場で修正する。
   主張内容の変更を伴う指摘（考察の妥当性、結論の強さ）は修正せず報告に留める。
4. 新たな残課題を見つけたら GitHub Issue 化を提案する。

## チェックリスト

### 前提（仕様・ドキュメント由来の記述）
- [ ] 上流ドキュメント・コード由来の記述に出典タグ [n] が付いており、参考文献章に対応があるか
- [ ] スコープ宣言（評価対象 API・対象外機能）が README とレポート本文で一致しているか
- [ ] 「ドキュメント記載の事実」と「コードからの推定」が区別されているか（推定には 🔎 考察ブロック等のマーカー）

### 結果（実測値）
- [ ] 数値に実行条件（データ・サンプル数・seed・ε などのパラメータ・依存バージョン）が紐付いているか
- [ ] 同じ指標が複数ページに出る場合、値が一致しているか（本体 ↔ 追加実験 ↔ 手法解説ページ）
- [ ] 図（figures/）と本文の数値・キャプションが一致しているか
- [ ] 単一シードの値を断定的に扱っていないか（揺れの注記、または mean±std の併記)

### 考察
- [ ] 各考察が、どの結果（実験 A/B/... や表番号）を根拠にしているか明示されているか
- [ ] 結果が支持する範囲を超えた一般化をしていないか（「本実験条件では」等の限定があるか）
- [ ] 機構・手法間の差を論じる際、ばらつき（std）に対して差が有意かに触れているか
- [ ] 構成要素のアルゴリズム固有の深掘り（手法 A が B を上回る理由・構造的限界・計算コスト等）が本体に残らず個別ページに分離され、本体は検証対象主軸の要約＋リンクになっているか

### 課題
- [ ] 残課題章の項目が GitHub Issues と対応しているか（漏れ・重複）
- [ ] 対応済みの課題に ✅ と残作業が追記されているか
- [ ] 適用範囲の限界（単一データセット・特定バージョン等）が結論・アブストラクトに反映されているか

### 文体（AI 臭の除去・technical/academic register 版）

[stop-ai-slop-jp](https://github.com/iKora128/stop-ai-slop-jp)（Daichi Nagashima）から、技術・学術文書に効く観点だけを採用する。
**論文体ではブログ向けルールは適用しない**（後述「適用しない」を参照）。原則は「撒くな（1〜2回は自然、頻出すると臭う）」。

- [ ] **false agency**: モノが人間の動詞をしていないか。「データが示している／課題が浮き彫りになる／結果が物語る」→「測定値を見ると〜／〜で課題に気づいた／〜から〜が分かる」。誰が・何を測ったかに直す。
- [ ] **翻訳調動詞**: 指している・示唆している・収斂する・浮かび上がる の多用。具体動詞へ。
- [ ] **vague declaratives（根拠なき抽象評価）**: 「本質的な問題」「構造的な課題」「示唆に富む」で止まらず、何が・なぜ・どの数値かに降りる。降りられない文は削る。
- [ ] **強い評価＋根拠なし**: 「画期的」「非常に有効」を数値・条件なしで使わない。
- [ ] **二項対比の多用**: 「A ではなく B だ」「単なる A でなく B」→ 直接「B だ」。違いを論じる時だけ具体的に残す。
- [ ] **3 項目並列の惰性**: 「3 つの観点／原則」が水増しでないか（RQ や本当に 3 つある実験条件は正当。揃えるための 3 つ目を疑う）。
- [ ] **過剰カタカナ／横文字メタファー**: 技術用語はそのままでよいが、普通の日本語に戻せる語（コンテキスト→文脈、アライメント→整合、プライオリティ→優先）は戻す。
- [ ] **リチュアル化した disclaimer**: 各節末に同じ保険文を反復していないか。限界は §考察・限界に集約し、本文での繰り返しを削る（学術文書では限界節自体は必須なので消さない）。
- [ ] **記号アーティファクト**: 全角ダッシュ ── は読点/コロン/改行へ。`**` 残骸、コロン後の半角スペース、中黒 A・B・C の機械的並列、装飾絵文字（🚀🎯✨💡）の均等撒布を検索→除去。
- [ ] **メタ構造宣言**: 「以下の 3 つの観点から説明する」「ここからが本題」等、見出しで分かる予告を本文で繰り返していないか。
- [ ] **リズムの均一さ**: 全段落が同じ長さ・同じ文末で揃いすぎていないか（音読で息が続かない/単調な箇所を探す）。

**適用しない（論文体では不適切なので除外）**: 毒・皮肉・自虐の追加、一人称「自分は／俺は」、中間温度の口語（「微妙」「ヤバい」「よく分からん」）、伝聞調（「らしい」「だそうです」）の多用、アカデミック自称の全廃（本追試・本研究・本稿は学術文書では許容）。これらは stop-ai-slop-jp がブログ・エッセイ向けに挙げる項目であり、検証レポートには持ち込まない。

## 出力形式

```

### Issue 化候補
- ...
```
